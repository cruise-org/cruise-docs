## Why Cruise?

<br />
<br />

Your terminal deserves better. Container management tools are either too simple or buried in a web UI. Cruise lives where you already work.

<br />

<CardGrid stagger>
  <Card title="Live container inspection" icon="magnifier">
    Real-time stats,  mounts, and more — all without running a single CLI command.
  </Card>
  <Card title="Streaming logs" icon="document">
    Search, and filter logs from any container in real time. Follow mode built in.
  </Card>
  <Card title="Artifact lifecycle management" icon="list-format">
    Browse, pull, tag, and remove images, volumes, and networks from a unified view.
  </Card>
  <Card title="Keyboard centric" icon="laptop">
    Every action is a keypress away. Fully navigable without ever touching a mouse.
  </Card>
  <Card title="Vulnerability scanning" icon="warning">
    Optional Trivy and Grype integration — scan images for CVEs directly from the TUI.
  </Card>
  <Card title="Fully configurable" icon="setting">
    Custom keybinds, colour themes, and runtime behaviour via a simple TOML(soon to be YAML) config file.
  </Card>
  <Card title="Event stream" icon="information">
    Watch live runtime events as they happen — container starts, stops, network connects, and more.
  </Card>
  <Card title="Runtime agnostic" icon="puzzle">
    Docker today, Podman and more on the roadmap. One interface for every container runtime.
  </Card>
  <Card title="Zero dependencies" icon="rocket">
    Single static binary. No runtime, no daemon, no setup. Works anywhere containers run.
  </Card>
</CardGrid>
